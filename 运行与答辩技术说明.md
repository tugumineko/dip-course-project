# 运行与答辩技术说明

> 用途：现场演示 App、打开幻灯片、回答老师关于模块目的和技术实现的提问。

## 一、如何打开项目 App

### 1. 推荐启动方式

在 PowerShell 中进入项目目录：

```powershell
cd E:\数字图像处理\项目
E:\miniconda3\envs\ai-service\python.exe 代码\main.py
```

说明：

- 必须使用 `ai-service` 环境，因为该环境已包含 OpenCV、PyTorch、torchvision、tkinter 等依赖。
- 不建议使用系统默认 `python`，它不是本项目的主运行环境。
- 程序启动后窗口标题为“老照片修复与风格化迁移纪念”。

### 2. 备用启动方式

如果已经激活 conda 环境：

```powershell
conda activate ai-service
cd E:\数字图像处理\项目
python 代码\main.py
```

### 3. 打开幻灯片

直接用浏览器打开：

```powershell
E:\数字图像处理\项目\演示文稿.html
```

或在文件管理器中双击 `演示文稿.html`。

## 二、现场演示建议流程

### Step 1：启动 App

运行：

```powershell
E:\miniconda3\envs\ai-service\python.exe 代码\main.py
```

### Step 2：打开一张老照片

菜单：

```text
文件 → 打开
```

推荐选择：

```text
素材\真实老照片\R.jpg
```

如果不想展示真实家庭照片，也可以用：

```text
素材\合成老照片\test_old.jpg
```

### Step 3：展示家庭老照片修复模块

左侧选择“家庭老照片修复”中的功能，建议演示顺序：

1. **温和修复**：展示默认修复效果，说明适合大多数轻度退化老照片。
2. **泛黄褪色修复**：展示色彩校正，说明保留少量暖色记忆感。
3. **扫描颗粒修复**：展示去噪但保留轮廓。
4. **人像清晰化**：展示人物轮廓增强。

每次点击后，看右侧“当前结果”与左侧“原图”的对比。

### Step 4：展示纪念版生成模块

在修复结果基础上，选择“纪念版生成”：

1. **复古相册风**：适合家庭相册保存。
2. **暖色胶片风**：适合怀旧生活照。
3. **纪念明信片风**：适合展示和礼物卡片。
4. **淡彩手绘风**：适合柔和纪念图。

建议现场重点演示“纪念明信片风”，视觉效果最容易让老师理解“纪念化输出”。

### Step 5：展示 VGG 高级自定义艺术风格

选择：

```text
纪念版生成 → 自定义艺术风格迁移（VGG）
```

推荐参数：

- 风格图：`素材\风格图\warm_family_memory_style.jpg`
- 风格强度：3
- 迭代步数：100 或 150
- 处理尺寸：512

说明：

- 该功能会调用预训练 VGG19，运行时间比传统模板长。
- 如果现场时间紧，可以只展示预生成结果：`结果\gallery\vgg_家庭暖调风格迁移.jpg`。

### Step 6：导出对比图

菜单：

```text
文件 → 导出对比图
```

用途：生成“原图 + 当前结果”的并排图，可用于报告、PPT 或现场保存展示。

## 三、模块目的说明

### 1. 家庭老照片修复模块

**目的**：解决家庭旧照常见质量退化，让照片更清楚、更干净、更适合保存。

包含 6 个模式：

| 模式 | 解决的问题 | 技术思路 |
| :--- | :--- | :--- |
| 温和修复 | 轻微发黄、灰蒙、颗粒 | 轻度 NLM、黄化校正、CLAHE、亮度 USM |
| 泛黄褪色修复 | 偏黄、偏棕、褪色 | LAB 空间限幅去黄、L 通道增强、轻微饱和恢复 |
| 黑白旧照增强 | 灰度层次弱、边缘不清 | 灰度去噪、CLAHE、S 曲线、暖纸色 |
| 扫描颗粒修复 | 背景颗粒、暗部脏点 | YCrCb 分离、色度强去噪、亮度弱去噪、边缘保护 |
| 人像清晰化 | 人物轮廓略模糊 | 双边基础层、细节层、梯度 mask、亮度 gate |
| 折痕划痕辅助修复 | 白线、裂痕、局部破损 | top-hat / black-hat 掩膜 + 用户掩膜 + inpaint |

### 2. 纪念版生成模块

**目的**：将修复结果进一步做成适合保存、展示或赠送的纪念图。

包含 4 个模板：

| 模板 | 效果 | 技术思路 |
| :--- | :--- | :--- |
| 复古相册风 | 旧相册照片质感 | 低饱和、暖色曲线、暗角、米色边框 |
| 暖色胶片风 | 怀旧胶片质感 | S 曲线、暖高光、轻微颗粒、暗角 |
| 纪念明信片风 | 卡片式展示图 | 米色画布、白边、投影、标题区域 |
| 淡彩手绘风 | 柔和绘画感 | 双边滤波、Canny 浅线稿、色彩融合 |

### 3. VGG 自定义艺术风格模块

**目的**：满足用户上传任意风格图的高级个性化需求。

技术路线：

```text
内容图 + 风格图 → VGG19 特征提取 → 内容损失 + Gram 矩阵风格损失 → L-BFGS 优化输出图像素
```

特点：

- 使用预训练 VGG19；
- 不训练网络；
- 支持任意风格图；
- 运行时间比传统模板更长；
- 适合作为进阶功能展示。

### 4. 高级算法功能

**目的**：支持课程展示、参数实验和进一步精修。

包含：

- 基础操作：灰度化、旋转、缩放；
- 对比度增强：直方图均衡、CLAHE、Gamma、白平衡；
- 去噪：均值、中值、高斯、双边、NLM；
- 锐化：Laplacian、Sobel、USM；
- 图像恢复：运动模糊、逆滤波、维纳滤波、inpaint；
- 频域处理：傅里叶频谱、低通、高通；
- 小波去噪；
- 分割和形态学。

## 四、老师可能提问与回答

### Q1：你的项目解决了什么实际问题？

A：项目面向家庭老照片，解决发黄、褪色、灰蒙、颗粒、模糊、划痕等质量问题，并进一步生成适合保存和展示的纪念版图像。

### Q2：为什么要设计多个修复模式？

A：家庭老照片的退化类型不同。泛黄照片重点是颜色校正，黑白旧照重点是灰度层次，扫描件重点是颗粒噪声，人像照片重点是轮廓清晰。不同模式对应不同需求。

### Q3：温和修复具体做了什么？

A：它先做轻度 NLM 去噪，再做黄化校正和 CLAHE 局部增强，最后在亮度通道做温和 USM 锐化。参数比较保守，目标是自然改善。

### Q4：泛黄褪色修复为什么用 LAB 空间？

A：LAB 空间把亮度和颜色分开。L 通道控制明暗，a/b 通道控制颜色。老照片黄化主要体现在 b 通道偏移，所以在 LAB 中更容易做受控校正。

### Q5：扫描颗粒修复为什么分离 YCrCb？

A：Y 通道主要表示亮度和结构，Cr/Cb 表示色度。色度噪声可以较强去除，亮度通道要保守处理以保留边缘。这样比直接整图强去噪更适合人物照片。

### Q6：人像清晰化有没有用深度学习？

A：没有。它使用传统图像处理：双边滤波得到基础层，计算细节层，再用梯度 mask 和亮度 gate 控制锐化区域，避免暗部噪声被放大。

### Q7：折痕划痕能完全自动修复吗？

A：不能保证完全自动。系统用 top-hat 和 black-hat 辅助提取细线划痕，同时支持用户加载掩膜补充破损区域，最后用 inpaint 修补。

### Q8：纪念版生成和普通滤镜有什么区别？

A：这些模板是围绕家庭照片展示需求设计的。比如明信片风会生成米色画布、白边、投影和标题区域，目标是形成可保存、可展示的纪念图，而不仅是改变颜色。

### Q9：VGG 风格迁移的作用是什么？

A：它是高级自定义功能。用户可以上传任意风格图，系统用预训练 VGG19 提取内容特征和风格特征，通过 Gram 矩阵表示风格，不训练网络，只优化输出图像素。

### Q10：为什么传统模板和 VGG 都保留？

A：传统模板速度快、稳定、适合家庭纪念主流程；VGG 适合用户想要更自由的自定义艺术效果。二者满足不同层次需求。

### Q11：为什么某些模式 PSNR 不一定最高？

A：PSNR 衡量像素级接近程度，但色彩校正、锐化、纪念模板都会主动改变像素。真实老照片没有标准答案，所以还需要看主观自然度、清晰度和纪念效果。

### Q12：项目有哪些工程细节？

A：主要有三个：第一，Windows 中文路径下 OpenCV 读写失败，所以封装了 `utils/imgio.py`；第二，VGG 风格迁移放后台线程运行，避免 GUI 卡死；第三，所有算法输入输出统一为 ndarray，便于 GUI 调用和脚本批量生成展示图。

## 五、现场演示注意事项

1. **先演示传统修复和纪念模板**，因为这些功能响应快、稳定。
2. **VGG 风格迁移可作为高级功能演示**，如果现场时间紧，展示预生成结果即可。
3. **不要承诺自动修复所有划痕**，表述为“辅助检测 + 用户掩膜 + inpaint”。
4. **不要只强调 PSNR**，要说明真实老照片更看自然度和视觉改善。
5. **如果 App 未响应**，优先等待 VGG 任务完成；传统滤镜一般不会卡住。
