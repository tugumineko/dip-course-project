# 老照片修复与风格化迁移纪念 — 项目设计稿

> 课程：ECNU SEI《数字图像处理_2026》（曹桂涛 教授）期末项目  
> 形态：单人 · tkinter 桌面程序 · 经典数字图像处理 + Gatys 风格迁移  
> 运行环境：conda `ai-service`（`E:\miniconda3\envs\ai-service\python.exe`，GPU 就绪）

---

## 1. 项目定位

本项目面向家庭老照片的修复与纪念化需求，提供一套从“照片导入”到“修复输出”的完整桌面应用。

一句话定位：

> 用户导入一张发黄、灰蒙、有颗粒噪声、轻微模糊或带划痕的老照片，系统通过“一键修复 + 针对性精修 + 艺术纪念版生成”的流程，帮助用户得到可保存、可展示、可纪念的结果。

项目不以“算法清单”为第一叙事，而以真实照片问题为入口。完整数字图像处理工具箱保留为高级模式，用于课堂展示、参数实验和进一步精修。

---

## 2. 应用场景

### 2.1 目标用户

- 想修复家中老照片的普通用户；
- 需要演示经典数字图像处理算法效果的课程答辩场景；
- 希望把修复照片做成相册封面、纪念海报或礼物卡片的用户。

### 2.2 典型流程

```text
打开老照片 → 判断退化问题 → 一键温和修复 → 针对性精修 → 艺术纪念版生成 → 保存 / 导出对比图
```

用户可以只走“一键修复”快速得到结果，也可以继续进入高级工具箱逐步调节。

---

## 3. 老照片问题与处理策略

| 老照片问题 | 用户看到的现象 | 处理策略 | 系统功能 |
| :--- | :--- | :--- | :--- |
| 发黄偏色 | 整体偏黄、偏棕、肤色不自然 | 估计通道均值并做白平衡 | 白平衡 |
| 低对比 / 灰蒙 | 画面发灰、人物轮廓弱 | 拉开局部或全局灰度分布 | CLAHE、直方图均衡、Gamma |
| 颗粒噪声 | 胶片颗粒、扫描噪点、椒盐点 | 邻域统计或自相似去噪 | 中值、双边、NLM |
| 轻微模糊 | 边缘不清、人脸细节弱 | 细节层增强或退化模型恢复 | USM、维纳滤波 |
| 划痕 / 折痕 | 白线、裂痕、局部破损 | 掩膜标出破损区域并自动填补 | inpaint |
| 纪念化输出 | 想做相册封面、海报或礼物图 | 保留内容结构并迁移艺术纹理 | Gatys 风格迁移 |

## 4. 功能需求

### 4.1 普通修复入口

| 功能 | 说明 | 优先级 |
| :--- | :--- | :--- |
| 打开 / 保存图片 | 支持 jpg、png、bmp、tif 等格式 | P0 |
| 原图 / 当前结果并排预览 | 直观看修复前后变化 | P0 |
| 一键温和修复 | 自动执行去噪、白平衡、增强、轻微锐化 | P0 |
| 去黄偏色 | 快速调用白平衡 | P0 |
| 提亮与增强 | 快速调用 CLAHE | P0 |
| 去颗粒噪声 | 快速调用 NLM 或中值去噪 | P0 |
| 增强清晰度 | 快速调用 USM 锐化 | P0 |
| 划痕/折痕修复 | 选择掩膜，白色区域表示待修复区域 | P1 |
| 艺术纪念版生成 | 选择风格图，生成艺术风格版本 | P0 |
| 导出对比图 | 保存“原图 + 结果”的并排图 | P1 |

### 4.2 高级算法工具箱

保留完整经典图像处理功能，用于课堂展示和精修：

- 基础操作：灰度化、转彩色、旋转、缩放；
- 增强：直方图均衡、CLAHE、正规化、分段线性、对数、Gamma、白平衡；
- 去噪：均值、中值、高斯、双边、非局部均值；
- 锐化：Laplacian、Sobel、USM；
- 分割/边缘：固定阈值、Otsu、自适应阈值、Canny；
- 形态学：腐蚀、膨胀、开运算、闭运算、梯度、顶帽；
- 图像恢复：运动模糊模拟、加噪、逆滤波、维纳滤波；
- 频域：傅里叶频谱、理想/巴特沃斯低通、高通；
- 小波：软阈值去噪。

## 5. 系统架构

```text
┌─────────────────────────────────────────────┐
│        应用界面层：tkinter                  │
│  修复向导 · 高级工具箱 · 并排预览 · 导出      │
└──────────────────┬──────────────────────────┘
                   │ ndarray 进出
┌──────────────────▼──────────────────────────┐
│        算法核心层：dip/ + style/             │
│  增强 · 去噪 · 锐化 · 恢复 · 频域 · 小波       │
│  一键修复流水线 · Gatys 风格迁移              │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  OpenCV · NumPy · SciPy · PyWavelets · torch │
└─────────────────────────────────────────────┘
```

设计原则：

- **应用入口优先**：左侧顶部先展示“老照片修复向导”。
- **算法可独立测试**：所有算法函数统一为 `ndarray -> ndarray`。
- **高级功能不删除**：课程展示需要的工具箱完整保留，但不抢占应用主线。
- **耗时任务后台运行**：风格迁移用后台线程 + 进度回调。
- **中文路径安全**：全项目图像 I/O 统一使用 `utils/imgio.py`。

---

## 6. 一键修复流水线

默认流水线：

```text
可选划痕修复 → 去噪 → 白平衡 → CLAHE 增强 → USM 轻微锐化
```

顺序理由：

1. 先去噪，避免后续增强和锐化放大噪点；
2. 再白平衡，校正发黄和通道偏移；
3. 再增强，提升局部对比度；
4. 最后锐化，补偿去噪带来的轻微模糊。

默认目标是“稳定、自然、不过度处理”，不是单一指标最大化。

---

## 7. 艺术纪念版生成

采用 Gatys 风格迁移方法：

- 使用 ImageNet 预训练 VGG19 作为特征提取器；
- 内容由深层特征图表示；
- 风格由多层特征的 Gram 矩阵表示；
- 直接优化输出图像素，最小化内容损失和风格损失。

应用解释：

- 修复版：保留真实记忆；
- 艺术版：用于纪念海报、相册封面、礼物卡片；
- 支持预设风格和自定义风格图。

工程约束：

- 处理长边默认 512px，可调至 720px；
- GPU 优先，CPU 兜底；
- 后台线程运行，主界面保持响应。

## 8. GUI 设计

### 8.1 主窗口结构

```text
┌───────────────────────────────────────────────────────────┐
│ 菜单：文件(打开/保存/导出对比)  编辑(撤销/重置)  查看/帮助   │
├──────────────┬────────────────────────────────────────────┤
│ 老照片修复向导│            预览区：原图 | 当前结果           │
│ ▸ 一键温和修复│   ┌──────────────┐   ┌──────────────┐       │
│ ▸ 去黄偏色    │   │   原图        │   │   当前结果     │       │
│ ▸ 提亮增强    │   └──────────────┘   └──────────────┘       │
│ ▸ 去颗粒噪声  ├────────────────────────────────────────────┤
│ ▸ 增强清晰度  │ 参数区：随所选功能显示滑块 / 下拉 / 复选框    │
│ ▸ 划痕修复    ├────────────────────────────────────────────┤
│ ▸ 艺术纪念版  │ 状态栏：当前操作 · 图像信息 · 进度提示        │
│ 高级工具箱    │                                            │
└──────────────┴────────────────────────────────────────────┘
```

### 8.2 交互约定

- 打开图片后，原图和当前结果并排显示；
- 点击无参数功能直接应用；
- 点击有参数功能，在下方参数区显示滑块、下拉框或复选框；
- 每次应用前入历史栈，支持 30 步撤销；
- 保存只保存当前结果，导出对比图会拼接原图和结果；
- 风格迁移在子窗口完成，迁移完成后回写主窗口当前结果。

## 9. 素材与评估

### 9.1 双轨素材

| 轨道 | 用途 | 做法 |
| :--- | :--- | :--- |
| 合成退化样张 | 量化评估 | 对干净图叠加噪声、低对比、偏色、轻微模糊和划痕 |
| 真实老照片 | 应用展示 | 使用家庭扫描件，展示主观改善 |

### 9.2 指标与主观评价

- 合成图使用 PSNR / SSIM，因为有干净原图；
- 真实图没有标准原图，用偏色、对比度、噪声、细节、自然度等维度评价；
- 报告中明确说明：PSNR 不完全等于主观质量，增强和白平衡可能降低像素级指标但改善观看效果。

## 10. 风险与对策

| 风险 | 影响 | 对策 |
| :--- | :--- | :--- |
| 风格迁移耗时导致 GUI 卡死 | 高 | 后台线程运行，主线程只更新进度 |
| 大图风格迁移显存不足 | 中 | 限制长边，默认 512px，最大 720px |
| 真实老照片效果不稳定 | 中 | 默认温和参数，展示合成量化 + 真实主观双轨结果 |
| 划痕修复需要掩膜 | 中 | 明确提示白色区域为待修补区域，演示用已知掩膜 |
| Windows 中文路径读写失败 | 高 | 使用 `np.fromfile` / `cv2.imdecode` 和 `cv2.imencode` / `tofile` 封装 |
| 现场 GPU 或环境异常 | 高 | 准备预渲染结果图，GUI 仍可展示经典修复功能 |
