## 1. 项目概述与定位

### 1.1 一句话定位

做一个**面向"老照片"的桌面图像处理工具**：用户导入一张褪色/有噪点/划痕/模糊的老照片，借助一套**经典数字图像处理算法**逐步或一键修复增强，并可把修复后的照片做**艺术风格迁移**留念。

### 1.2 为什么选它（动机，演讲稿要用）

- **贴近生活**：家里翻出的老照片普遍褪色、起噪、有折痕，"修复老照片"是人人能共情的真实需求。
- **课程覆盖最全**：一张老照片的"退化 → 恢复"恰好把第 2/3/4/5/6/7/8/9/10 章串成一条完整故事线，**第九章图像恢复**（逆滤波/维纳滤波）正好是课程难点也是亮点。
- **不依赖大算力、零训练**：经典修复用 OpenCV；进阶的风格迁移用 Gatys 优化法只借预训练 VGG19，符合"无训练时间"的约束。

### 1.3 难度档映射（给老师/助教看的"为什么是 1.3"）

| 难度档 | 本项目对应内容 |
| :--- | :--- |
| **系数 1（地基）** | 一套完整的图像处理系统：基础操作 + 对比度增强 + 平滑去噪 + 锐化 + 分割/边缘 + 形态学 + 图像恢复 + 频域处理，带可交互 tkinter 界面 |
| **+0.3（进阶项①）** | **图像风格迁移**（老师五选一中的第①项），Gatys 神经风格迁移，GPU 加速 |

> 注：本项目按 **1.3** 设计；"老照片修复"本身也具备 1.5"场景明确的实际应用"气质，若后期想冲 1.5，可在"上色 + 批量处理 + 修复报告"等方向加码（见 §14 可选加分项）。

## 2. 需求分析

### 2.1 目标用户与使用场景

- **用户**：想修复家里老照片的普通人 / 课程演示的老师助教。无图像处理专业背景。
- **典型场景**：
  1. 打开一张老照片 → 看到它的问题（噪点、偏暗、模糊、划痕）。
  2. 方式 A「一键修复」：点一个按钮，自动跑一条修复流水线，直接看前后对比。
  3. 方式 B「逐步精修」：在左侧功能面板里逐个尝试去噪/增强/锐化/修复，实时看效果，可撤销。
  4. 满意后 → 可选「艺术风格化」→ 选一张风格图，生成艺术版。
  5. 保存结果 / 导出"前后对比图"用于演示。

**手写算法清单（"关键几个手写"落地，含 OpenCV 对照验证）**：
1. **直方图均衡化**（第四章）：算直方图 → 归一化累积分布(CDF) → 映射。对照 `cv2.equalizeHist`。
2. **空域卷积 + 均值/Laplacian 核**（第六/七章）：手写滑窗卷积，演示邻域处理与核的作用。对照 `cv2.filter2D`。
3. **中值滤波**（第六章）：手写邻域取中位数，演示非线性滤波对椒盐噪声的效果。对照 `cv2.medianBlur`。
4. **Otsu 阈值**（第五章，可选第 4 个）：手写类间方差最大化求阈值。对照 `cv2.threshold(...,THRESH_OTSU)`。

> 手写版只需在**中小尺寸**上跑通并出对照图（证明理解原理）；界面实际处理大图时用 OpenCV 版保证速度。

## 5. 处理流水线与数据流

### 5.1 单步处理数据流

`原图(ndarray) → [选中的算法 + 参数] → 结果(ndarray) → 刷新预览 + 入历史栈`

- 维护 `original`（原图，永不变）、`current`（当前结果）、`history`（撤销栈）。
- 所有算法对 `current` 操作，结果回写 `current`；"重置"= `current ← original`。

### 5.2 「一键修复」流水线（FR-12）

按"老照片常见退化"的合理顺序：

```
中值/双边去噪(第六章) → 直方图均衡/伽马 提亮提对比(第四章)
→ USM 适度锐化(第七章) →（可选）维纳去模糊(第九章)
```

- 每步参数给"温和"的默认值，避免过冲。
- 一键完成后展示前后对比；用户仍可在此基础上继续精修。

## 6. 算法实现策略（手写 vs OpenCV）

| 类别 | 策略 |
| :--- | :--- |
| I/O、几何、形态学、Canny、双边、inpaint | **直接 OpenCV**（稳定、快） |
| 直方图均衡、空域卷积(均值/Laplacian)、中值、Otsu | **手写 numpy** + 旁附 OpenCV 结果**对照图**（证明理解） |
| 傅里叶频域 | `numpy.fft`（手写频域滤波器：理想/巴特沃斯低通高通） |
| 维纳/逆滤波 | numpy 按公式实现（第九章核心，手写更显理解） |
| 风格迁移 | torch + 预训练 VGG19（不手写网络，但手写损失与优化循环） |

> 对照图思路：手写结果 vs OpenCV 结果 vs 两者差值图，差值接近全黑即证明实现正确——这本身就是答辩里很有说服力的一页。

## 7. 风格迁移模块设计（1.3 核心）

### 7.1 方法：Gatys 优化法（A Neural Algorithm of Artistic Style, 2015）

**核心思想**（演讲稿要讲清）：
- 用 **ImageNet 预训练的 VGG19** 当"特征提取器"，**不训练它**。
- **内容**由深层特征图表示；**风格**由各层特征的 **Gram 矩阵**（通道间相关性）表示。
- 从内容图（或噪声）出发，**直接对输出图的像素做梯度下降**，最小化：
  `总损失 = α·内容损失 + β·风格损失`
- 几百步迭代后，输出图既保留老照片的内容结构，又呈现风格图的笔触色彩。

**为什么满足约束**：唯一的"模型"是预训练 VGG19（torchvision 自动下载），**没有任何训练/数据集**；任意内容 + 任意风格。

### 7.2 参数与默认值

| 参数 | 默认 | 说明 |
| :--- | :--- | :--- |
| 内容层 | `conv4_2` | 经典选择 |
| 风格层 | `conv1_1,2_1,3_1,4_1,5_1` | 多尺度风格 |
| 内容权重 α | 1 | — |
| 风格权重 β | 1e6（相对值，界面给"风格强度"滑块） | 调风格化程度 |
| 迭代步数 | 200–300（界面可调，给"质量/速度"档） | GPU 上几十秒 |
| 处理分辨率 | ≤ 720px（长边） | 控显存与时间 |
| 优化器 | L-BFGS 或 Adam | L-BFGS 收敛快 |
| 设备 | `cuda`（自动回退 `cpu`） | — |

### 7.3 工程要点
- **后台线程**跑优化，界面显示进度（每 N 步回调更新进度条），结束后回主线程刷新预览（见 §13 R-9）。
- 灰度老照片先转 3 通道再迁移；输出可选"保留原图亮度只迁移色彩/笔触"作为对比项。
- 预设几张风格图（梵高《星空》、水墨、油画、素描、浮世绘）放 `素材/风格图/`，也支持用户上传任意风格图。

## 9. 演示素材方案（双轨）

| 轨道 | 用途 | 做法 |

## 10. 评估与指标
- **PSNR**（numpy 即可）：合成样张"修复结果 vs 干净原图"，量化修复效果，做成"参数 → PSNR"小实验，答辩加分。
