## 一、项目概述

### 1.1 项目定位

本项目是一个面向**家庭老照片修复与纪念化输出**的桌面应用。它不把用户暴露在大量算法参数前，而是针对家庭老照片的常见特点，提供几类定制修复模式：温和修复、泛黄褪色修复、黑白旧照增强、扫描颗粒修复、人像清晰化、折痕划痕辅助修复。

修复完成后，系统还提供传统图像处理实现的纪念版模板，包括复古相册风、暖色胶片风、纪念明信片风、淡彩手绘风。VGG19 Gatys 风格迁移仍保留，但定位为“自定义艺术风格迁移”高级功能，而不是家庭纪念输出的默认方式。

### 1.2 设计动机

家庭老照片的真实问题通常是混合退化，而不是单一算法问题。例如一张照片可能同时存在发黄、褪色、低对比、扫描颗粒、轻微模糊和折痕。普通用户很难判断该使用中值滤波、CLAHE、NLM 还是维纳滤波。

因此本项目的设计重点从“工具箱式算法罗列”转向“场景化定制方案”：

```text
选择照片类型 / 修复目标 → 系统调用固定处理流水线 → 用户查看前后对比 → 生成纪念版
```

这样既保留数字图像处理算法的可解释性，也更符合家庭老照片修复的实际应用场景。

## 二、家庭老照片特点分析

| 场景 | 典型现象 | 修复目标 | 定制模式 |
| :--- | :--- | :--- | :--- |
| 普通家庭旧照 | 发黄、灰蒙、轻微噪点 | 自然改善，不过度处理 | 温和修复 |
| 泛黄褪色照片 | 颜色偏黄/偏棕，饱和度低 | 校正黄化，保留少量暖色 | 泛黄褪色修复 |
| 黑白旧照 | 灰度层次弱、边缘不清 | 增强明暗层次和轮廓 | 黑白旧照增强 |
| 扫描颗粒照片 | 暗部和背景有明显颗粒 | 减少颗粒但保留人像边缘 | 扫描颗粒修复 |
| 家庭人像/合影 | 人物轮廓弱，脸部不够清晰 | 提升中等边缘，避免噪声放大 | 人像清晰化 |
| 折痕划痕照片 | 白线、裂痕、局部破损 | 自动辅助掩膜 + inpaint 修复 | 折痕划痕辅助修复 |
| 纪念展示 | 想做相册封面、明信片、礼物图 | 生成温和、怀旧、家庭感模板 | 纪念版生成 |

该表决定了系统的主入口。用户不再先选算法，而是先选照片场景或想达到的修复目标。

## 三、定制修复算法设计

新增定制算法模块位于 `代码/dip/family_restore.py`。这些函数不是单个基础算法，而是围绕家庭照片特点组合出来的固定流水线。

### 3.1 温和修复

适合多数家庭旧照，目标是“看起来更干净、更自然，但仍保留旧照质感”。

```text
轻度 NLM 去噪 → 黄化校正与亮度增强 → 低强度 CLAHE → 亮度通道温和 USM
```

设计要点：

- NLM 强度较低，避免把人物脸部抹平；
- 黄化校正不完全去暖色，保留旧照片的记忆感；
- USM 只在亮度通道上做，并使用噪声门限，避免锐化暗部噪点。

### 3.2 泛黄褪色修复

老照片常见整体偏黄、偏棕。如果直接使用 Gray World 白平衡，可能把照片调得过冷，失去旧照氛围。因此该模式使用 LAB 空间进行限幅校正：

```text
BGR → LAB → 估计 b 通道黄蓝偏移 → 限幅去黄 → L 通道 CLAHE → 轻微饱和度恢复
```

核心策略是“去掉过量黄化，保留少量暖色”。这样结果不像现代手机照片，而像被修复过的旧照片。

### 3.3 黑白旧照增强

黑白照片的重点不是色彩，而是灰度层次和轮廓清晰度。

```text
灰度化 → 轻度 NLM → CLAHE → S 曲线增强 → 亮度细节锐化 → 可选暖纸色
```

输出可保留为黑白，也可加入轻微暖纸色，模拟老相纸观感。

### 3.4 扫描颗粒修复

扫描件的噪声常集中在背景和暗部。如果直接强去噪，人物边缘容易被抹掉。本模式采用亮度 / 色度分离：

```text
YCrCb 分离 → 色度通道强去噪 → 亮度通道弱去噪 → Canny 边缘保护 → 权重融合
```

边缘区域更多保留原图，平坦区域更多使用去噪结果，从而减少颗粒同时保留轮廓。

### 3.5 人像清晰化

家庭老照片通常以人物为主体，但项目不引入人脸识别模型，而用传统边缘增强实现保守清晰化：

```text
亮度通道 → 双边滤波得到 base → detail = 原亮度 - base → 梯度 mask + 亮度 gate → 局部增强细节
```

它只增强中等边缘，暗部噪声区域会被亮度门限抑制，避免“清晰化”变成“噪声放大”。

### 3.6 折痕划痕辅助修复

原系统只支持用户加载掩膜，应用感较弱。新模式增加自动辅助掩膜：

```text
灰度图 → top-hat / black-hat 提取亮线和暗线 → 形态学细线增强 → Otsu 阈值 → 与用户掩膜合并 → inpaint
```

该方法不承诺完全自动识别所有破损，但可以辅助发现明显细线划痕；用户也可加载白色掩膜补充破损区域。

## 四、纪念版生成设计

新增纪念模板模块位于 `代码/dip/commemorative.py`。主流程使用传统图像处理滤镜，不依赖深度学习，保证速度快、结果稳定、风格贴合家庭照片。

| 模板 | 处理方式 | 适用场景 |
| :--- | :--- | :--- |
| 复古相册风 | 低饱和、暖色曲线、柔和暗角、米色相纸边框 | 家庭相册、长辈旧照 |
| 暖色胶片风 | 柔和 S 曲线、暖高光、轻微颗粒、轻暗角 | 家庭合影、生活照 |
| 纪念明信片风 | 米色画布、照片白边、投影、英文纪念标题 | 礼物卡片、汇报展示 |
| 淡彩手绘风 | 双边平滑、浅线稿、低强度色彩提升 | 柔和纪念图、手绘感输出 |

### 4.1 为什么主流程不用名画风格

《星空》《神奈川冲浪里》能展示风格迁移技术，但它们和家庭老照片的情感语境不完全一致。家庭纪念更适合温和、怀旧、相册式的视觉语言。因此主流程改为传统图像处理纪念模板，VGG 风格迁移作为高级自定义功能保留。

### 4.2 VGG 自定义艺术风格迁移

Gatys 方法仍然保留在 `代码/style/neural_style.py`：

- 使用预训练 VGG19 提取内容和风格特征；
- 内容由深层特征图表示；
- 风格由多层 Gram 矩阵表示；
- 优化输出图像素，不训练模型；
- GUI 中命名为“自定义艺术风格迁移（VGG）”。

本次新增一张更贴近家庭氛围的暖调风格参考图 `素材/风格图/warm_family_memory_style.jpg`，用于替代名画风格作为主展示案例。

│   │   └── ...                 # 高级算法工具箱

### 5.2 GUI 结构

GUI 左侧分成三层：

1. **家庭老照片修复**：温和修复、泛黄褪色修复、黑白旧照增强、扫描颗粒修复、人像清晰化、折痕划痕辅助修复；
2. **纪念版生成**：复古相册风、暖色胶片风、纪念明信片风、淡彩手绘风、自定义艺术风格迁移（VGG）；

这样普通用户一打开程序先看到定制修复模式，老师答辩时也能看到底层经典算法的完整性。

## 六、实验与评估

### 6.1 展示素材

本项目生成了三类展示图：

- **定制修复图**：每种家庭修复模式一张前后对比；
- **纪念模板图**：复古相册、暖色胶片、纪念明信片、淡彩手绘；
- **高级功能图**：CLAHE、NLM、USM、Canny、频谱、维纳滤波、小波等压缩展示。

### 6.2 量化结果

合成退化样张仍用于 PSNR / SSIM 评估。结果中，中值版一键修复指标最高，这是因为合成退化中椒盐/划痕类噪声较明显；定制修复模式更偏真实照片主观效果，默认保守、自然、不过度锐化。

| 方法 | PSNR (dB) | SSIM |
| :--- | :--- | :--- |
| 退化老照片（基线） | 16.15 | 0.4007 |
| 温和修复 | 16.64 | 0.4476 |
| 泛黄褪色修复 | 16.49 | 0.4209 |
| 扫描颗粒修复 | 16.24 | 0.4170 |
| 人像清晰化 | 15.97 | 0.3987 |
| 一键修复（默认） | 16.02 | 0.4231 |
| 一键修复（中值） | 18.06 | 0.5964 |
| 中值滤波 k=3 | 17.37 | 0.5743 |

### 6.3 指标解释

PSNR 衡量像素级接近程度，但老照片修复不能只看 PSNR：

- 色彩校正会改变像素值，但视觉上更自然；
- 纪念模板会主动改变色调、边框和风格，不适合用 PSNR 衡量；
- 人像清晰化可能增加边缘对比，PSNR 不一定升高；
- 真实老照片没有干净原图，只能用主观维度评估。

因此项目采用“合成图量化 + 真实图主观对比”的双轨评估。

## 七、技术难点

### 7.1 定制修复不能过度处理

家庭老照片的价值在于“旧照质感”。如果增强过强，会变得像现代滤镜。解决方法是在修复模式中使用低强度参数、亮度通道处理和边缘/亮度门限。

### 7.2 黄化校正不能完全去暖色

简单白平衡可能让老照片偏冷。项目在 LAB 空间中对 b 通道做限幅校正，并保留少量暖色。

### 7.3 颗粒修复要保护人像边缘

直接强去噪会抹掉人物轮廓。项目在 YCrCb 空间中分离亮度和色度，并用 Canny 边缘权重融合原图和去噪图。

### 7.4 划痕自动检测存在局限

top-hat / black-hat 能辅助发现细线划痕，但对复杂破损不稳定。因此系统保留手动掩膜入口，将自动检测定位为“辅助”。

### 7.5 VGG 风格迁移耗时较长

Gatys 方法需要逐图优化，512px 约几十秒。因此它不作为默认纪念模板，而作为高级自定义功能。主纪念模板均由传统图像处理实现，可即时反馈。

## 八、总结

本项目最终定位为一个**面向家庭老照片的定制化数字图像处理应用**。相比单纯的算法工具箱，它的改进在于：

1. 以家庭老照片场景为入口，提供固定修复模式；
2. 用传统图像处理构建快速稳定的纪念版模板；
3. 保留 VGG19 Gatys 风格迁移作为高级自定义功能；
4. 高级算法工具箱仍完整保留，支撑课程知识展示和参数实验。

通过这套设计，项目既能体现数字图像处理课程的算法实践，也更贴近真实家庭老照片修复与纪念化输出需求。
